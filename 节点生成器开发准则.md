# 标准节点生成器开发准则

> 用于开发新语言节点生成器（Go / Java / C++ / Shell 等）

---

## 一、节点目录结构

每个节点在 `项目/nodes/` 下必须有如下结构：

```
nodes/
└── node_{lang}_{name}/
    ├── config.json          # 节点配置（必需）
    ├── start.bat             # Windows 启动脚本（必需）
    ├── start.sh              # Linux/macOS 启动脚本（必需）
    ├── listener.{ext}        # 监听器入口（必需，负责持续运行并处理输入）
    ├── main.{ext}            # 主处理入口（可选，单次处理逻辑）
    ├── requirements.txt      # 依赖声明（如有）
    ├── .gitignore            # 忽略编译产物/日志
    └── logs/                 # 运行日志目录（自动创建）
        └── listener.log
```

---

## 二、config.json 规范

```json
{
  "node_name": "节点名称",
  "language": "语言标识（python/rust/go/java/cpp/shell）",
  "output_type": "输出类型（json/text）",
  "listen_upper_file": "",
  "input_type": "json",
  "description": "节点描述"
}
```

**语言标识**必须与 `ui/creators/node_creator_manager.py` 中注册的 key 一致。

## 三、start.bat 强制规范（Windows）

### 必须支持的参数

| 参数 | 行为 |
|------|------|
| `--no-pause` | 静默模式：不输出 UI 信息，不 `pause`，供 GUI 调用 |
| 无参数 | 交互模式：显示完整信息，`pause` 等待用户 |

### 必须实现的功能

1. **环境自愈**：检测运行依赖是否存在，缺失时自动修复
2. **后台启动**：使用 `start /b` 启动监听程序，不阻塞脚本
3. **PID 记录**：启动后写入 `.pid` 文件，供 GUI 进程健康检测

### 模板

```bat
@echo off
setlocal enabledelayedexpansion
if not "%1"=="--no-pause" (
    cls
    chcp 65001 >nul
    echo ======================================
    echo        BNOS {Lang} Node Starter
    echo ======================================
    echo.
)
cd /d "%~dp0"

REM === 环境自愈 ===
REM 检测 {运行时/编译器/venv} 是否存在，缺失则自动构建/修复
REM ...

REM === 后台启动 ===
if not "%1"=="--no-pause" (
    echo 🔧 后台启动监听程序...
    start /b "" {启动命令}
) else (
    start /b "" {启动命令} >nul 2>&1
)

REM === 写入 PID ===
REM 等待进程启动
timeout /t 1 /nobreak >nul
REM 用 tasklist 获取进程 PID（替换 {exe_name} 为实际进程名）
for /f "tokens=2" %%i in ('tasklist /fi "imagename eq {exe_name}" /nh 2^>nul') do echo %%i > .pid

if not "%1"=="--no-pause" (
    echo ✅ 监听程序已在后台运行 (PID 已写入 .pid)
    pause
)
```

### 关键实现细节

- **`start /b ""`**：必须加 `""` 避免 Windows 把第一个参数当窗口标题
- **`timeout /t 1`**：等待 1 秒让进程启动，否则 `tasklist` 查询不到
- **`2^>nul`**：在 bat 文件内必须写成 `2^>nul`，命令行用 `2>nul`
- **PID 文件只写一行一个数字**：`echo 1234 > .pid`

## 四、start.sh 强制规范（Linux/macOS）

### 模板

```bash
#!/bin/bash
cd "$(dirname "$0")"
NO_PAUSE=false
[ "$1" = "--no-pause" ] && NO_PAUSE=true

# === 环境自愈 ===
# ...

# === 后台启动 + PID ===
nohup {启动命令} > /dev/null 2>&1 &
echo $! > .pid
[ "$NO_PAUSE" = false ] && echo "✅ 监听程序已在后台运行 (PID: $(cat .pid))" && read -p "按回车键退出..."
```

### 关键实现细节

- **`nohup ... &`** 搭档写：后台运行且不受终端关闭影响
- **`echo $! > .pid`**：`$!` 是上一个后台进程的 PID
- **`[ "$NO_PAUSE" = false ]`**：条件执行 UI 输出，静默模式不打扰

## 五、listener 程序规范

监听程序是节点的核心，必须持续运行（不主动退出）。

### 最低功能要求

1. 读取本节点 `config.json` 获取 `listen_upper_file` 路径
2. 监听上游节点的 `output.json` 文件变化
3. 收到新数据时调用处理逻辑
4. 将处理结果写入本节点 `output.json`
5. 日志写入 `logs/listener.log`

### 自愈机制

- 启动时检测运行环境是否完整，缺失则自动修复
- 输出文件路径不存在时自动创建目录
- 异常时不崩溃，记录日志后继续监听

## 六、与 NodeCreatorManager 集成

### 注册新语言

在 `ui/creators/node_creator_manager.py` 的 `_register_default_creators` 中添加：

```python
self._register("{lang_key}", {
    "display_name": "{显示名称}",
    "creator_func": lambda name: create_{lang}_node(name),
})
```

### 调用 tools 生成脚本

creator 函数应：

1. 导入对应的 tools 脚本（如 `from tools.{lang}_create_node import generate`）
2. 调用生成函数创建节点目录和所有文件
3. 返回 `True` / `False`

## 七、测试清单

新语言节点生成器完成后，必须通过以下全部测试：

| 测试项 | 步骤 | 预期结果 |
|--------|------|----------|
| 创建节点 | GUI → 新建节点 → 选择语言 | 节点目录完整，config.json 正确 |
| 编译/构建 | 首次启动节点 | 环境自愈生效，自动构建成功 |
| 前台启动 | 双击 `start.bat` | 显示完整 UI，`pause` 等待 |
| 后台启动 | GUI 启动节点 | 进程在后台运行，UI 显示 ● 运行中 |
| PID 记录 | 启动后检查 `.pid` 文件 | 文件存在，内容为有效 PID |
| 进程检测 | 重启 GUI 后检查状态 | 自动恢复为 ● 运行中 |
| 进程退出 | 手动 kill 进程后等 3 秒 | UI 自动变为 ○ 已停止 |
| 停止节点 | GUI 停止节点 | `.pid` 文件被删除，进程终止 |
| 日志输出 | 查看 `logs/listener.log` | 正常记录运行日志 |

## 八、禁止事项

| 禁止 | 原因 |
|------|------|
| 前台阻塞 `{程序}` | 不自启后台，GUI 无法检测 |
| 不写 `.pid` 文件 | 无法跨会话恢复状态 |
| `pause` 不加 `--no-pause` 判断 | GUI 调用时卡住 |
| `stdout=subprocess.PIPE` 不读取 | 缓冲区满后进程卡死 |
| 编译产物混入提交 | 用 `.gitignore` 排除 `target/`、`venv/`、`.exe` 等 |
